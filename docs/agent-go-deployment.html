1. Objectif

Ce document explique le deploiement complet de l agent Go sur postes clients Windows:

  • Build du binaire
  • Signature optionnelle
  • Installation locale en service Windows
  • Deploiement massif a distance
  • Deploiement via GPO Startup Script
  • Verification, troubleshooting et rollback

3. Prerequis

3.1 Poste de build

  • Windows 10/11 ou Windows Server
  • Go SDK 1.22+
  • PowerShell 5.1+
  • Acces au backend (HTTP/HTTPS)

3.2 Postes clients

  • Droits administrateur local pour installer le service
  • Sortie reseau vers backend sur port 8000 ou via reverse proxy 443
  • Execution des scripts PowerShell autorisee (ou bypass process)

3.3 Remoting (si deploiement massif)

  • WinRM / PowerShell Remoting actif
  • Acces admin share \\PC\C$
  • Compte admin de domaine ou local admin sur cibles

5. Build du binaire

  1. Ouvrir PowerShell dans agent-go
  2. Lancer le script de build
Set-Location D:\projet-dev\It-monutoring\agent-go
.\build_agent.ps1

Resultat attendu: agent-go/deploy/it-agent.exe

6. Signature du binaire (optionnel mais recommande)

Script: agent-go/deploy/sign_agent.ps1

Set-Location D:\projet-dev\It-monutoring\agent-go\deploy
$cred = Get-Credential
.\sign_agent.ps1 -PfxPath "C:\certs\codesign.pfx" -SigningCredential $cred
Utiliser un certificat de signature de code valide entreprise. Ne pas distribuer de mot de passe PFX en clair.

7. Installation locale (poste client)

  1. Copier le dossier deploy sur le poste client
  2. Ouvrir PowerShell en Administrateur
  3. Lancer l installation service
Set-Location C:\Temp\agent-go\deploy
.\install_service.ps1

Verification service

Get-Service ITMonitoringGoAgent
Get-Content C:\ProgramData\ITMonitoringAgent\agent.log -Tail 50

Desinstallation

.\uninstall_service.ps1

8. Deploiement massif (remote)

Script: agent-go/deploy/deploy_bulk.ps1

  1. Renseigner les machines dans computers-example.txt (1 machine par ligne)
  2. Lancer le deploiement depuis un poste admin
Set-Location D:\projet-dev\It-monutoring\agent-go\deploy
.\deploy_bulk.ps1 `
  -ComputerListPath .\computers-example.txt `
  -BackendUrl "http://srv-it:8000/assets/scan" `
  -SkipOffline

9. Deploiement via GPO (Startup Script)

9.1 Preparation partage

  • Creer un partage central, ex: \\DOMAIN\netlogon\it-agent-go
  • Y copier: it-agent.exe, config.json, gpo_startup_install.ps1

9.2 Lier script dans GPO

  1. Group Policy Management
  2. Computer Configuration -> Policies -> Windows Settings -> Scripts (Startup)
  3. Ajouter gpo_startup_install.ps1 avec arguments:
-SourceSharePath "\\DOMAIN\netlogon\it-agent-go" -BackendUrl "http://srv-it:8000/assets/scan"

9.3 Log GPO

Sur poste client: C:\ProgramData\ITMonitoringAgent\gpo-startup.log

10. Variables d environnement supportees

VariableDescription
ASSET_BACKEND_URLURL backend si config absent
ASSET_BACKEND_PROXYProxy sortant HTTP/HTTPS
ASSET_AGENT_LOG_PATHChemin log local

11. Flux envoyes au backend

  • /assets/scan - inventaire machine + logiciels
  • /security/posture - posture securite
  • /metrics/resources - metriques CPU/RAM/Disque
  • /network/telemetry - ports ouverts + logs reseau

L agent envoie aussi: agent_source=go, agent_version, agent_id pour dual-run.

12. Validation post-installation

  1. Service up:
Get-Service ITMonitoringGoAgent
  1. Logs agent:
Get-Content C:\ProgramData\ITMonitoringAgent\agent.log -Tail 80
  1. Backend dual-run:
GET /agents/dual-run/health
GET /agents/dual-run/compare/{serial_number}

13. Troubleshooting

13.1 Service ne demarre pas

  • Verifier Event Viewer System
  • Verifier presence du binaire dans C:\ProgramData\ITMonitoringAgent
  • Verifier droits locaux et AV/EDR

13.2 Pas de remontee backend

  • Verifier backend_url
  • Verifier route reseau/firewall/proxy
  • Verifier logs backend

13.3 Remoting bulk KO

  • Activer WinRM: Enable-PSRemoting -Force
  • Verifier pare-feu remoting
  • Tester Invoke-Command manuellement

14. Bonnes pratiques securite

  • Signer le binaire avant deploiement massif
  • Utiliser HTTPS/TLS vers backend en production
  • Deployer via compte admin limite et journaliser les actions
  • Conserver un plan rollback (uninstall_service.ps1)
